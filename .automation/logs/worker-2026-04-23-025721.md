- REP-168 production change: `isSyncing` visibility changed from `private` to `private(set)` so tests can observe the flag from the same module under `@testable import`. The flag is set to `true` at the start of `syncFromIMessage` and reset to `false` in a `defer` block.
- `testIsSyncingTrueWhileSyncing` required careful synchronization in Swift 6 + `@MainActor` context. Polling `vm.isSyncing == true` deadlocked because `contacts.ensureAccess()` hops to the generic executor AFTER setting `isSyncing = true`, meaning `recentThreads()` is not yet called when the test first sees the flag.
