## Actions this fire

1. **P0 deflake — runtime test failure under midnight clock time:** discovered
   `IMessageChannelFormatTimeTests.testFormatRelativeUsesHoistedYesterdayLabel`
   was failing this fire with `"Thu" != "Yesterday"`. Root cause: the test
   used `Date() - 25h` to construct a "clearly yesterday" timestamp, but at
   the actual fire start time (00:11 EDT 2026-05-09), 25h ago is 23:11
   2026-05-07 — Thursday, the day BEFORE yesterday. `isDateInYesterday`
   correctly rejected it and the formatter rendered the weekday-short
   branch. Fix: switched to `Calendar.startOfDay(for: Date()) - 1 day + 12h`
   so the test produces yesterday-noon regardless of current clock time.
   Same pattern as the sibling
   `testFormatRelativeYesterdayBucketReturnsYesterdayLiteral` already in
   the file. → main `aae88f6`.

2. **P5 cross-module pin — AppleScript application target equality:**
   `AppleScriptMessageReader.recentChats()` and `.messagesForChat()` emit
   `tell application "Messages"` AppleScript inline literals at lines 110
   and 166. `IMessageSender.appleScriptApplicationTarget` hoists the same
   target string to a named constant. Each side was independently pinned
   (`"Messages"` literal in both files' tests), but no test asserted their
   cross-equality — a refactor that renamed the constant (e.g. to
   `"MessagesApp"`) would update IMessageSender's pin while silently
   leaving the reader's inline literals at `"Messages"`, splitting the
   read and send paths onto different AppleScript targets. Added
   `testReaderScriptUsesSameApplicationTargetAsSender` in
   `AppleScriptMessageReaderTests.swift` — re-runs both reader scripts
   and asserts each contains
   `tell application "<IMessageSender.appleScriptApplicationTarget>"`.
   +1 test (1895 → 1896). → main `0d667f2`.

3. **P8 doc bump:** `agents.md: bump test count header 1895 → 1896` →
   main `d316473`.

4. **P8 gotcha addition — clock-sensitive Date-delta test pattern:** the
   bug fixed in action 1 is a pattern worth documenting. Added a new
   AGENTS.md gotcha entry directly above the `swift test` Contacts-XPC
   gotcha, explaining that `Date() - N hours` for sub-day offsets is
   fragile near midnight and DST boundaries, and pointing at the safe
   `Calendar.startOfDay(for:)` pattern used in the deflake fix and the
   sibling robust test. Future fires touching time-bucket tests now have
   a one-paragraph reference. → main `ea7d4c5`.

5. **P5 theme pin — `accentInk` literal hex:** an audit of every
   `static let` in `Sources/ReplyAI/` against test references surfaced
   `Theme.Color.accentInk` as the only public-but-unpinned theme token.
   It's the inverse text color stacked on the brand accent (chartreuse-
   lime) — every primary CTA's label ("Get started", "Send", "Open
   inbox"), the menu-bar `R` glyph, and unread-count badges all use
   this color. Drift either drops contrast below WCAG AA (washed-out
   ink) or pops too hard (jet-black-on-chartreuse). Added
   `testAccentInkLiteralIsPinnedDeepInk` asserting
   `String(describing: Theme.Color.accentInk) == "#0A0B0DFF"`. The
   sibling `testSurfaceBg1LiteralIsPinnedDarkInbox` already pins the
   same hex against `bg1`, but the two tokens are conceptually distinct
   (text-on-accent vs primary surface) — a designer who intentionally
   splits them needs each pinned independently. +1 test (1896 → 1897).
   → main `0128700`.
