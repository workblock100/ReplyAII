## Approach

Rewrote `AttributedBodyDecoder` to use the correct typedstream primitive-string
type tag (0x2B, "+") rather than scanning for an NSString class name. Key insight:
class-name C strings in typedstream use a plain length-byte encoding without the
0x2B prefix, so the first 0x2B in the archive body is reliably the NSString (or
NSMutableString) text payload — regardless of whether the root object is NSString,
NSMutableAttributedString, or any subclass. This sidesteps the substring-matching
bug in the old code where "NSString" appeared inside "NSMutableString" and the
scan started from the wrong position.

Added consecutive-duplicate filtering for encoders that emit the same string twice
(NSMutableAttributedString + its NSMutableString ivar sometimes produce two copies).
