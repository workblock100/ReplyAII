## Root Cause (REP-199)

`BlockingMockChannel` is `@unchecked Sendable` — its `recentThreads()` method hops off the main actor to the cooperative pool thread. Three fields (`recentThreadsCallCount`, `blocking`, `pending`) were plain `var` behind `NSLock`, but the computed-property reads on those fields took the lock separately from the writes, creating a TOCTOU window. Swift 6.3 on macOS 26 surfaced this as non-deterministic crashes.

Additionally, `InboxViewModelAutoPrimeTests`, `InboxViewModelThreadSelectionTests`, and `InboxViewModelReselectTests` all created `InboxViewModel` without isolated `RulesStore` or `SearchIndex` instances, so concurrent test runs shared the production file path and in-memory SQLite state, introducing interference between tests.

## Fix

1. **`BlockingMockChannel`**: Replaced all mutable fields with `Locked<T>` backing stores (existing codebase pattern). All reads and writes now go through `withLock`, eliminating the data race.

2. **Test isolation**: Added `makeRules()` helper (temp-directory `RulesStore` with unique UUID path) and `SearchIndex(databaseURL: nil)` (in-memory SQLite) to `InboxViewModelAutoPrimeTests`, `InboxViewModelThreadSelectionTests`, and `InboxViewModelReselectTests`. Each test method now creates a fully isolated VM.

3. **REP-201**: Updated test count in AGENTS.md from `463 tests` to `493 tests` (grep-accurate: `grep -c "func test" Tests/ReplyAITests/*.swift` = 493). SHA `904b0e7` was already absent from AGENTS.md (contained `7512321`); no SHA change needed.
