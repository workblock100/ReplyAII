### Lessons for next fire
- **Never poll on `pgrep -f "swift test"`** — the parent dies before the
  xctest child finishes, AND stale xctest from prior sessions can satisfy
  the match. Use `swift test` inline as a foreground command and trust the
  bash timeout, or use the task framework's natural completion signal
  (TaskOutput notification arrives when the bash exits).
- **Always `pkill -9 -f xctest` at fire-start** as a sanity step. Cheap, and
  it preempts the exact failure mode this fire hit.
